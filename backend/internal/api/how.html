How it works

From a beacon to a live floor plan

RoomPulse turns Bluetooth proximity into live room presence. Here's the path data takes today — and where it's headed as we move from a prototype to real hardware and real bookings.

The live data flow · today
Room beacon
Broadcasts the room's identity (UUID · major · minor) over Bluetooth, continuously.
an iPhone today
Your phone
The RoomPulse app ranges nearby beacons and decides which room you're in — even with the app closed.

RoomPulse backend
Receives presence heartbeats, tracks live occupancy and history, and drives check-in / out.
Go · SQLite
Dashboard & floor
Occupancy lights up in real time on this site — rooms, people, and a live map.
you're here
Bookings are simulated for now. Rooms and reservations come from a built-in mock of Zoom Workspace, so the whole flow runs with no credentials. Presence, occupancy and history are fully real.
Where it's going · roadmap
● Live now
Phase 1
Phone beacons + presence

The end-to-end prototype, working today.

  • An iPhone broadcasts each room's beacon
  • Phones auto check in / out by proximity
  • Live occupancy, history & floor map
Next
Phase 2
ESP32 beacons

Replace the phone-broadcaster with real hardware.

  • A tiny always-on ESP32 iBeacon per room
  • No phone needed to broadcast — runs 24/7
  • Same UUID · major · minor scheme, zero app changes
Planned
Phase 3
Real Zoom / Academy data

Swap the mock for live bookings.

  • Live Zoom Workspace Reservation API (or Academy's system)
  • Real rooms & reservations
  • Check-in / out written back to the source
